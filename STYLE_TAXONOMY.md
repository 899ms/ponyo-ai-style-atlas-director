# STYLE TAXONOMY — V2 Director / IP Atmosphere System

V2 的分类逻辑不再是“50 个风格平铺”。  
新的分类由两层组成：

1. **高价值层：知名导演 / 经典 IP / 艺术大师氛围转译库**
2. **支撑层：通用视觉风格库**

Skill 应优先使用高价值层；只有当用户没有明显导演 / IP / 大师氛围需求时，再使用通用风格库。

---

- 某个导演 / IP 氛围在视觉上有什么具体特征？

## 3. 推荐策略

当用户给出主题时，优先判断：

1. 是否适合某个导演 / IP 氛围？
2. 是否有商用安全风险？
3. 是否需要转译为泛化视觉 DNA？
4. 是否需要加入通用支撑风格增强画面？

示例：

```text
用户：做一个治愈乡村短片。
不要只推荐“清新森系”。
应推荐：日系手绘治愈动画风 65% + 清新森系 20% + 水彩插画质感 15%。
```

```text
用户：做一个儿童品牌 3D 广告。
不要只推荐“3D 卡通”。
应推荐：温润家庭向 3D 动画电影感 60% + 3D 盲盒风 25% + 商业广告布光 15%。
```

```text
用户：做一个城堡童话视觉。
不要只推荐“梦幻城堡”。
应推荐：华丽童话城堡动画感 60% + 古典绘本装饰 25% + 黄金时刻电影光影 15%。
```
